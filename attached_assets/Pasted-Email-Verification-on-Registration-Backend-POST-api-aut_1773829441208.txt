Email Verification on Registration

  - Backend: POST /api/auth/register now returns { requiresVerification: true, email, simulatedCode } instead of
  auto-login
  - Backend: New endpoints: POST /api/auth/verify-email, POST /api/auth/resend-code
  - Frontend: After registration, shows a verification code input screen with the simulated code displayed (since no
  real email service)
  - Database: New verification_codes table

  3. Forgot Password with Email Verification

  - Backend: POST /api/auth/forgot-password generates a 6-digit code, POST /api/auth/reset-password validates and resets
  - Frontend: "Reset Password?" link below sign-in form, 2-step flow (enter email → enter code + new password)

  4. Change Name & Password in Profile Settings

  - Backend: PUT /api/auth/settings accepts { fullName?, currentPassword?, newPassword? }
  - Frontend: New "Settings" tab in student dashboard with change name, change password (requires current password), and
   shows user ID

  5. Company Admin Settings

  - Database: Added bio, contact_email, contact_phone, location, working_hours, requirements_per_country columns to
  companies table
  - Backend: GET/PUT /api/company/settings endpoints
  - Frontend: Full settings panel in Company Admin with fields for: name, bio, contact details, location, working hours,
   requirements per country, pricing, specialized countries

  6. Super Admin User Management

  - Backend: POST /api/admin/users (create), DELETE /api/admin/users/:id (delete), PUT /api/admin/users/:id/role (change
   role/assign company)
  - Backend: All admin routes now require super_admin role via middleware
  - Frontend: Users table with search, add user modal, delete button, change role modal (with company assignment for
  company_admin)

  7. Auto-Generated 6-Character User ID

  - Database: user_code column on users table (unique, e.g. A3F9K2)
  - Generated automatically on registration (after email verification)
  - Existing users backfilled with codes
  - Displayed in student dashboard settings

  8. Super Admin Search by User ID

  - Backend: GET /api/admin/users/search?code=XXXXXX
  - Frontend: Search bar with monospaced input in Super Admin, shows highlighted result card
