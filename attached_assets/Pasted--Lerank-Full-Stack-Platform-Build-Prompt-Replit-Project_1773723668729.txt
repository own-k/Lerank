# Lerank — Full-Stack Platform Build Prompt (Replit)

## Project Overview

**Lerank** is a trust-first marketplace that connects international students with verified education consultants. It solves one critical problem: students pay thousands of dollars to consultants who rely on verbal promises and sometimes scam them. Lerank introduces **real-time process monitoring** and **escrow-protected payments** so students can track every step of their application and get a 100% refund if the consultant doesn't deliver.

**Tech Stack:** Next.js 14 (App Router), Supabase (Auth, Database, Storage, Realtime), Tailwind CSS, Framer Motion, TypeScript

**Domain:** lerank.com (or lerank.io)

## Database Schema (Supabase/PostgreSQL)

```sql
-- Users table (extends Supabase auth.users)
CREATE TABLE profiles (
  id UUID PRIMARY KEY REFERENCES auth.users(id),
  full_name TEXT NOT NULL,
  email TEXT NOT NULL,
  role TEXT NOT NULL CHECK (role IN ('student', 'company_admin', 'super_admin', 'consultant')),
  avatar_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Student profiles
CREATE TABLE student_profiles (
  id UUID PRIMARY KEY REFERENCES profiles(id),
  ielts_score DECIMAL(2,1),
  sat_score INTEGER,
  toefl_score INTEGER,
  gpa DECIMAL(3,2),
  gpa_scale DECIMAL(2,1) DEFAULT 4.0,
  consulting_budget TEXT,
  education_budget TEXT,
  preferred_countries TEXT[],
  preferred_major TEXT,
  degree_level TEXT CHECK (degree_level IN ('bachelor', 'master', 'phd')),
  additional_notes TEXT
);

-- Companies
CREATE TABLE companies (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  logo_url TEXT,
  description TEXT,
  website TEXT,
  is_active BOOLEAN DEFAULT true,
  rating DECIMAL(2,1) DEFAULT 0,
  students_helped INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Company-country requirements
CREATE TABLE company_country_requirements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id UUID REFERENCES companies(id),
  country TEXT NOT NULL,
  min_ielts DECIMAL(2,1),
  min_sat INTEGER,
  min_gpa DECIMAL(3,2),
  tuition_range TEXT,
  available_majors TEXT[],
  deadline_info TEXT,
  success_rate DECIMAL(4,1),
  UNIQUE(company_id, country)
);

-- Services offered by companies
CREATE TABLE services (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id UUID REFERENCES companies(id),
  name TEXT NOT NULL,
  description TEXT,
  price DECIMAL(10,2) NOT NULL,
  currency TEXT DEFAULT 'USD',
  countries TEXT[],
  degree_levels TEXT[]
);

-- Milestone templates
CREATE TABLE milestone_templates (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  company_id UUID REFERENCES companies(id),
  name TEXT NOT NULL,
  milestones JSONB NOT NULL -- [{name, order, description}]
);

-- Engagements (student hiring a company)
CREATE TABLE engagements (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID REFERENCES profiles(id),
  company_id UUID REFERENCES companies(id),
  service_id UUID REFERENCES services(id),
  status TEXT DEFAULT 'active' CHECK (status IN ('active', 'completed', 'disputed', 'cancelled')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Milestones (per engagement)
CREATE TABLE milestones (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  engagement_id UUID REFERENCES engagements(id),
  name TEXT NOT NULL,
  description TEXT,
  status TEXT DEFAULT 'not_started' CHECK (status IN ('not_started', 'in_progress', 'under_review', 'completed')),
  order_index INTEGER NOT NULL,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  completed_at TIMESTAMPTZ
);

-- Transactions
CREATE TABLE transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  engagement_id UUID REFERENCES engagements(id),
  student_id UUID REFERENCES profiles(id),
  company_id UUID REFERENCES companies(id),
  amount DECIMAL(10,2) NOT NULL,
  escrow_fee DECIMAL(10,2), -- 5%
  platform_fee DECIMAL(10,2), -- 3%
  net_amount DECIMAL(10,2),
  currency TEXT DEFAULT 'USD',
  escrow_transaction_id TEXT, -- From Escrow.com API
  status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'funded', 'in_escrow', 'released', 'refunded', 'disputed')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Reviews
CREATE TABLE reviews (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  student_id UUID REFERENCES profiles(id),
  company_id UUID REFERENCES companies(id),
  engagement_id UUID REFERENCES engagements(id),
  rating INTEGER CHECK (rating BETWEEN 1 AND 5),
  comment TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable RLS on all tables
ALTER TABLE profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE student_profiles ENABLE ROW LEVEL SECURITY;
ALTER TABLE companies ENABLE ROW LEVEL SECURITY;
ALTER TABLE engagements ENABLE ROW LEVEL SECURITY;
ALTER TABLE milestones ENABLE ROW LEVEL SECURITY;
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;
```

## Additional Features (My Suggestions)

1. **In-App Messaging:** Real-time chat between student and consultant using Supabase Realtime. Messages stored in a `messages` table linked to engagements.

2. **Document Tracking:** Upload area for each milestone where consultants can attach drafts (SOP, recommendation letters, etc.) and students can review/approve. Use Supabase Storage.

3. **Email Notifications:** Send emails via Resend or Supabase Edge Functions when:
   - A milestone status changes
   - Payment is received/released/refunded
   - A new message arrives
   - Application deadline is approaching

4. **Review & Rating System:** After an engagement completes, prompt the student to rate (1-5 stars) and review. Display aggregate ratings on consultant cards.

5. **Smart Matching Algorithm:** Based on student's IELTS, SAT, GPA, budget, country preferences, and major — score and rank consultants. Use a weighted scoring formula matching student data against company_country_requirements.

6. **Dispute Resolution Flow:** If a student raises a dispute, freeze the escrow. Super admin reviews evidence from both sides and decides: release to consultant or refund to student.

7. **Public Consultant Profiles:** /consultant/[slug] pages showing: company info, services, reviews, success stats, countries served. Helps with SEO.

8. **Multi-Language Support:** Since users are international students, add i18n support (start with English, Uzbek, Russian, Turkish, Korean).
